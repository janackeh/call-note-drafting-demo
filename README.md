# Call-Note Drafting Demo

An interactive, rule-based portfolio demo of an **AI-assisted call-note drafting workflow** for customer support operations.

This is the demo featured at [janackeh.me/demo](https://janackeh.me/demo). The full write-up lives in the [case study](https://janackeh.me/case-study/).

## What it demonstrates

- **PII detection & masking** — order numbers, emails, phone digits, and card-like numbers are detected and masked before drafting.
- **Identity substitution** — drafts are written from the rep's first-name perspective ("Janackeh"), not a generic "Agent," so notes read naturally.
- **Result-only refinement (v2)** — a second pass rewrites the summary to lead with the outcome, cutting filler while keeping ticket numbers intact.
- **Human-in-the-loop diff** — a "Show what I changed" view highlights word-level edits between the AI draft and the refined version.
- **Multi-scenario sample data** — five anonymized support scenarios (billing, shipping, technical, return, escalation).
- **Session cap** — limits drafts per session to reinforce that this is a portfolio demo, not a production tool.

## Important notes

- **Rule-based, not live AI.** Drafts are generated by deterministic rules over anonymized sample data — there are no LLM API calls, no backend, and no storage. This keeps the demo safe to share publicly and faithful to the real workflow's structure.
- **Anonymized.** All names, orders, and ticket numbers are illustrative.
- **Not connected to any live customer system.**

## Run it locally

No build step. Either open `index.html` directly, or serve the folder (recommended, so fonts and paths resolve cleanly):

```bash
python3 -m http.server 8000
# then visit http://localhost:8000
```

## Host it

Push to a GitHub repo and enable **Settings → Pages → Deploy from branch (main / root)**. It will be live at `https://<your-handle>.github.io/<repo>/`.
